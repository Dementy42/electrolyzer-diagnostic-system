## 🔌 ПОДКЛЮЧЕНИЕ ОБОРУДОВАНИЯ

### Схема подключения Arduino MEGA 2560

```
АНАЛОГОВЫЕ ВХОДЫ (ADC):
┌─────────────────────────────────────────────────────────┐
│ A0  → Датчик давления (PWM, 0-500 кПа)                 │
│ A1  → Дифманометр HT-1890 канал A (начало трубки)       │
│ A2  → Дифманометр HT-1890 канал B (конец трубки)        │
└─────────────────────────────────────────────────────────┘

ЦИФРОВЫЕ ВЫХОДЫ:
┌─────────────────────────────────────────────────────────┐
│ Pin 10 → Датчик температуры DS18B20 (OneWire)          │
│ Pin 22 → Пневмоклапан (через реле 24V)                 │
│ Pin 24 → Реле системы (управление включением)          │
│ Pin 13 → Status LED (встроенный LED)                    │
└─────────────────────────────────────────────────────────┘

ПИТАНИЕ:
┌─────────────────────────────────────────────────────────┐
│ 5V   → Датчики давления, DS18B20                        │
│ GND  → Общая земля для всех датчиков                    │
│ Vin  → USB питание или внешний блок питания             │
└─────────────────────────────────────────────────────────┘
```

### Подробное подключение датчиков

#### Датчик давления (PWM, 0-500 кПа):
```
Датчик давления → Arduino MEGA 2560
┌──────────────┐
│   RED   ─────→ 5V (Pin 5V)
│   BLACK ─────→ GND (Pin GND)
│   GREEN ─────→ A0 (Analog Input 0)
└──────────────┘
```

#### Дифманометр HT-1890 (Два датчика):
```
HT-1890 Канал A → Pin A1
HT-1890 Канал B → Pin A2

Каждый датчик подключить:
┌──────────────┐
│   RED   ─────→ 5V (Pin 5V)
│   BLACK ─────→ GND (Pin GND)
│   GREEN ─────→ A1 или A2 (Analog Input)
└──────────────┘
```

#### Датчик температуры DS18B20:
```
          DS18B20
         ┌─────┐
         │  1  │ GND  → GND
         │  2  │ DQ   → Pin 10 (через 4.7kΩ резистор к 5V)
         │  3  │ VDD  → 5V
         └─────┘

СХЕМА ПОДКЛЮЧЕНИЯ:
         5V
         │
      [4.7kΩ]
         │
   ┌─────┴─────┐
   │           │
Pin10 ────→ DQ DS18B20
   │           │
  GND ────→ GND DS18B20
   │           │
   └──────────┘
```

#### Пневмоклапан (управление через реле):
```
Pin 22 → Реле 5V → Реле 24V → Пневмоклапан 24V

Arduino Pin 22 ──→ Реле сигнал
Реле 24V+ ────────→ Пневмоклапан +
Реле 24V- ────────→ Пневмоклапан -
```

#### Реле системы:
```
Pin 24 → Реле главное питание системы

Arduino Pin 24 ──→ Реле сигнал
Реле выход ───────→ Главное питание системы
```

## 📲 ЗАГРУЗКА ПРОШИВКИ НА ARDUINO

### Шаг 1: Подключение Arduino к macOS

```bash
# Проверить доступные порты
ls /dev/cu.*

# Результат должен показать что-то вроде:
# /dev/cu.usbmodem14201
# /dev/cu.usbserial-0001
```

### Шаг 2: Загрузка прошивки

1. Откройте **Arduino IDE**
2. Откройте файл: **File → Open → `arduino_mega_firmware.ino`**
3. Выберите плату: **Tools → Board → Arduino MEGA or Mega 2560**
4. Выберите порт: **Tools → Port → `/dev/cu.usbmodem14201`** (ваш порт)
5. Нажмите кнопку **Upload** (стрелка)
6. Дождитесь завершения загрузки (3-10 секунд)

### Шаг 3: Проверка прошивки

1. Откройте **Serial Monitor** (Tools → Serial Monitor)
2. Установите скорость **9600 baud**
3. Вы должны увидеть приветствие системы и "Waiting for commands..."

## 🔧 TROUBLESHOOTING

### Проблема: Arduino не найдена

**Решение:**
```bash
# Проверить доступные порты
ls /dev/cu.*

# Результаты:
# Если ничего не видно - проверить USB кабель
# Если видны cu.* - запустить скрипт заново, он должен найти

# Если нужно указать порт вручную, откройте hydrogen_analyzer_system.py
# и измените строку:
# controller = SystemController(config=config, arduino_port="/dev/cu.usbmodem14201")
```

### Проблема: Serial ошибки при чтении

**Решение:**
```
1. Проверить скорость передачи (должна быть 9600)
2. Перезагрузить Arduino (нажать RESET кнопку)
3. Переподключить USB кабель
4. Проверить, что прошивка загружена (должно быть приветствие в Serial Monitor)
```

### Проблема: Датчик DS18B20 не читается

**Решение:**
```
1. Проверить подключение (Pin 10 + 4.7kΩ резистор к 5V)
2. Проверить в Arduino IDE Serial Monitor - должна быть информация о датчике
3. Переустановить библиотеку DallasTemperature
```

### Проблема: Некорректные значения давления

**Решение:**
```
1. Провести повторную калибровку (опция 1 в меню)
2. Проверить калибровочные кривые датчиков
3. Убедиться, что система находится в нормальном состоянии при калибровке
```

### Проблема: Не сохраняются данные

**Решение:**
```bash
# Проверить права доступа к папке data
ls -la /project/data/

# Если нет прав:
chmod 755 /project/data/

# Если нет папки:
mkdir -p /project/data/
```
