## 🎯 Ключевые возможности

### Логика диагностики
✅ **ПЕРЕГИБ ТРУБКИ**
- Легкий (1.5x - 2.0x отклонение ΔP)
- Средний (2.0x - 3.0x)
- Критический (>3.0x)

✅ **РАЗРЫВ ТРУБКИ**
- Микроразрыв (-0.3 кПа/с)
- Частичный (-2.0 кПа/с)
- Полный (система автоматически останавливается)

✅ **ТЕМПЕРАТУРНЫЙ КОНТРОЛЬ**
- Предупреждение при >40°C или <5°C
- Коррекция давления с учетом температуры

### Функции управления
- 🔧 Открытие/закрытие пневмоклапана
- 🔧 Управление реле системы
- 🔧 Автоматическая остановка при критической ошибке
- 🔧 Индикация LED статуса

### Сохранение данных
- 📊 **Text Log** - все события с временем и деталями
- 📊 **CSV** - табличные данные для анализа
- 📊 **Excel** - форматированный отчет с цветовым выделением

## 🚀 БЫСТРЫЙ СТАРТ (5 минут)

### Шаг 1: Установка зависимостей
```bash
pip install pyserial openpyxl
```

### Шаг 2: Загрузка прошивки на Arduino
1. Откройте Arduino IDE
2. File → Open → `arduino_mega_firmware.ino`
3. Tools → Board → Arduino MEGA or Mega 2560
4. Tools → Port → /dev/cu.usbmodem*
5. Upload (нажать кнопку Upload)

### Шаг 3: Запуск Python приложения
```bash
cd /project/
python3 hydrogen_analyzer_system.py
```

### Шаг 4: Использование
```
Меню:
1. Калибровка системы (ОБЯЗАТЕЛЬНО ПЕРВЫМ)
2. Мониторинг 60 сек (нормальный режим)
3. Интенсивный мониторинг 30 сек (10 Гц)
4. Одна диагностика
5-8. Управление системой
9. Сохранить данные
0. Выход
```

## 🔌 Подключение оборудования

### Аналоговые входы (ADC)
- **A0** → Датчик давления (0-500 кПа)
- **A1** → Дифманометр HT-1890 (начало)
- **A2** → Дифманометр HT-1890 (конец)

### Цифровые входы/выходы
- **Pin 10** → DS18B20 (температура, OneWire)
- **Pin 22** → Пневмоклапан (реле 24V)
- **Pin 24** → Реле системы
- **Pin 13** → Status LED

**Все схемы подключения есть в README.md со схемами!**

## 🆘 Troubleshooting встроен

**Общие проблемы и решения:**

1. Arduino не найдена → `ls /dev/cu.*`
2. Sensor не читается → Проверить питание 5V
3. Data не сохраняется → `chmod 755 /project/data/`
4. Serial ошибки → Перезагрузить Arduino
5. Калибровка не работает → Убедиться в стабильности системы
